# roleplay

[English version →](README_EN.md)

roleplay 是一个纯技能型 Hanako 插件。它不改工具权限，不改记忆策略，也不引入额外运行时能力；它只负责一件事：让对话在用户主动开启后进入深度扮演，并持续推进场景与对话。

推荐使用场景：

- 用户明确说“开启扮演模式”
- 用户想要语C、酒馆、OC 房间、沉浸式剧本感的对话
- 用户希望所有行为、语言、节奏和格式都更贴合角色人设
- 用户还希望系统能主动引导、补问和续聊，而不是只做静态扮演

推荐触发口令：

- 开启扮演模式
- 进入扮演模式
- 开始扮演
- 关闭扮演模式
- 退出扮演模式

建议让用户提供的场景信息：

- 角色
- 场景
- 关系
- 叙事视角
- 语气 / 强度
- 边界 / 禁区

这个插件的默认风格是“深度扮演”而不是普通口吻切换：

- 所有行为、语言、节奏、格式都优先服务角色人设
- 输出尽量保持沉浸，不主动跳出角色做说明
- 默认采用动作 + 对白 + 轻量场景补充的写法
- 动作标记可使用 `*动作*` 或 `【动作】`，保持统一即可
- 用户指定语C / 酒馆 / 剧本 / 长文 / 轻文时，优先遵从用户偏好
- 如果用户要求退出扮演，再回到普通助手口吻

这个插件也负责“继续对话”而不只是“扮演”：

- 每轮回复尽量带着一个可继续的钩子，像提问、选择、动作、转折或邀请
- 如果用户只给了一个启动信号，先接住，再把话题往前推
- 如果场景信息不完整，优先用最少的问题补齐能推进戏的关键项
- 如果当前戏已经建立，顺着角色状态、关系变化和未完成事件继续往下写
- 如果用户暂时沉默，给出一句符合角色的过渡台词或动作，保持气氛流动
- 不要把每次回复都写成总结；要让对话自己往前长

安装后会加载的 skill：

- `roleplay`

## 使用方式

用户只要在对话里说：

> 开启扮演模式

然后给出角色和场景，或让系统沿着当前语境继续补全即可。

如果场景信息不完整，先用最短的问题补齐必要信息，再进入扮演。
