# Agent Fission · 技能路由

这是一个“分发判断”技能，不是创建技能。

它负责判断：一个新技能是该全员分发，还是只给少数匹配的代理，或者先保留不动。

## 适用场景

- 新技能刚安装完成，需要决定要不要给每个持久代理都启用。
- 用户明确说“把这个技能分给某些代理”或“看看哪些代理该装上”。
- 你已经知道技能的用途，但还没决定覆盖范围。

## 判断原则

- 通用基础能力、统一流程、跨代理都会用到的技能，优先考虑全员分发。
- 角色专用、项目专用、一次性练习、强上下文绑定的技能，只给匹配的代理。
- 某个代理自己学出来的私有自学技能，不要默认扩散到所有代理。
- 不确定时，先预览，再问用户，不要猜着全量写入。

## 工作流程

1. 先看技能是做什么的。
2. 调用 `agent-fission_sync_agent_skills`，先用 `mode: "preview"` 看当前代理列表、已启用情况，以及哪些代理目前能看到这个技能。
3. 如果技能是共享能力，就把所有符合条件的持久代理放进 `agentIds`，再用 `mode: "apply"` 分发。
4. 如果只适合一部分代理，就只把这些代理放进 `agentIds`。
5. 如果技能来源不明确，或明显是某个代理的私有自学技能，先问用户要不要推广。
6. 只追加这个技能，不要顺手改掉别的技能开关。

## 和新建 agent 的配合

如果用户要的是“新建代理时就带上这些技能”，不要走分发流程，直接用 `agent-fission_split_agent` 的 `enabledSkills` 参数。

## 快速判断

- 该全员分发：格式化、整理、通用工作流、基础护栏、跨角色都会用到的能力。
- 不该全员分发：角色专用、项目专用、测试用、私有自学技能。
- 拿不准：先 preview，再问用户。
